# Driver Drowsiness Detection System

A real-time computer vision system that monitors a driver's face via webcam or video file and raises an alarm when drowsiness or yawning is detected. Built with OpenCV, dlib facial landmarks, and an optional CNN classifier.

---

## Problem Statement

Drowsy driving is one of the leading causes of road accidents worldwide. According to NHAI data, fatigue-related accidents account for a significant proportion of highway fatalities in India. Existing commercial solutions (infrared cameras, steering sensors) are expensive and inaccessible. This project implements a low-cost, software-only drowsiness detection system using a standard webcam and computer vision algorithms.

---

## How It Works

1. **Face detection** — dlib's HOG-based detector locates the face in each frame
2. **68-point landmark extraction** — Facial keypoints isolate the eye and mouth regions
3. **Eye Aspect Ratio (EAR)** — Computed from 6 eye landmarks; drops sharply when eyes close
   ```
   EAR = (|p2−p6| + |p3−p5|) / (2 × |p1−p4|)
   ```
4. **Mouth Aspect Ratio (MAR)** — Detects prolonged mouth opening (yawning)
5. **CNN classifier** *(optional)* — A trained CNN validates eye state for higher accuracy
6. **Decision engine** — Sustained low EAR over N frames triggers alarm + red overlay

## Setup

### Requirements
- Python 3.8 or higher
- pip

### Step 1 — Clone the repository

```bash
git clone https://github.com/<your-username>/drowsiness-detector.git
cd drowsiness-detector
```

### Step 2 — Run one-time setup

```bash
python setup.py
```

This will:
- Install all pip dependencies from `requirements.txt`
- Download and extract the dlib landmark model (~99 MB)
- Generate `alarm.wav`
- Verify everything is working

> **Windows users:** dlib requires CMake and C++ build tools.
> Install from https://visualstudio.microsoft.com/visual-cpp-build-tools/ then run `pip install cmake dlib`.
>
> **Linux users:** Run `sudo apt-get install cmake build-essential` first.

### Step 3 — Generate the sample test video

```bash
python generate_sample_video.py
```

This creates `sample_video.mp4` — a 30-second synthetic video with simulated eye-closing and yawning events. No webcam needed.

---

## Running the Project

### Headless mode — video file input (for evaluators / servers)

```bash
python detector.py --input sample_video.mp4 --output result.avi --headless
```

Runs entirely in the terminal. No display, no webcam required.

### Live webcam mode (requires display)

```bash
python detector.py
```

Press `Q` to quit.

### All CLI options

```
usage: detector.py [-h] [-i INPUT] [-o OUTPUT] [--headless]
                   [--ear-thresh EAR_THRESH] [--ear-frames EAR_FRAMES]
                   [--mar-thresh MAR_THRESH] [--mar-frames MAR_FRAMES]

  -i, --input       Path to input video file (default: webcam)
  -o, --output      Path to save annotated output video (e.g. result.avi)
  --headless        Run without GUI — required on servers without a display
  --ear-thresh      EAR threshold below which eye is considered closing (default: 0.25)
  --ear-frames      Consecutive frames before drowsiness alarm fires (default: 20)
  --mar-thresh      MAR threshold above which a yawn is detected (default: 0.60)
  --mar-frames      Consecutive frames of yawning before alert (default: 15)
```

### Example commands

```bash
# Evaluate on sample video, save annotated output (evaluator command)
python detector.py --input sample_video.mp4 --output result.avi --headless

# Custom sensitivity
python detector.py --input sample_video.mp4 --headless --ear-thresh 0.22 --ear-frames 15

# Live webcam with output saved
python detector.py --output session.avi
```

## Training the CNN (optional)

Download the MRL Eye Dataset: http://mrl.cs.vsb.cz/eyedataset

Place images into:
```
dataset/
    open/    ← images of open eyes
    closed/  ← images of closed eyes
```

Then run:
```bash
python train_cnn.py
# or with custom options:
python train_cnn.py --data dataset --epochs 25 --batch 32
```

The best model is saved to `model/eye_model.h5` and automatically loaded by `detector.py`.

## Configuration

| Parameter | Default | Description |
|---|---|---|
| `EAR_THRESHOLD` | 0.25 | EAR below this = eye closing |
| `EAR_CONSEC_FRAMES` | 20 | ~0.67s at 30fps before alarm |
| `MAR_THRESHOLD` | 0.60 | MAR above this = yawning |
| `MAR_CONSEC_FRAMES` | 15 | Frames of yawning before alert |

## Course Concepts Applied

| Concept | Applied Where |
|---|---|
| Image processing (OpenCV) | Frame capture, grayscale conversion, contour drawing, video I/O |
| Face detection | dlib HOG-based frontal face detector |
| Facial landmark detection | dlib 68-point shape predictor |
| Geometric feature extraction | EAR and MAR formulas on landmark coordinates |

| Data augmentation | `ImageDataGenerator` in training pipeline |
| Real-time video processing | Frame-by-frame loop with threaded alarm |

## Limitations and Future Scope

- Glasses/sunglasses can occlude eye landmarks
- Performance degrades in low lighting
- Future work: PERCLOS metric, head pose estimation, IR camera support, TFLite mobile deployment
